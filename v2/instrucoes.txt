Como Executar e Apresentar o Trabalho
1 - Ordem de Execução:

Abra um terminal e rode: python servidor.py

Abra outro terminal e rode: python cliente.py (Este será o "Anfitrião/Jogador 0").

Abra mais terminais e rode python cliente.py para cada jogador adicional.

2 - Explicando os Requisitos Atendidos:

Estado Global: A classe EstadoJogo no protocolo.py é a "única verdade". O servidor a mantém e envia cópias (pickle) para os clientes.

Sincronização: Quando um jogador envia uma ação (JOGAR), o servidor processa, atualiza o estado e faz o broadcast para todos. Isso garante que todos vejam a mesma carta no topo.

Turnos: A variável jogador_atual e a verificação no servidor (if estado_jogo.jogador_atual == player_id) impedem que alguém jogue fora de hora.

Duplas: A lógica de Duplas está implementada visualmente no cliente (mostra quem é o parceiro) e na lógica de espera (aguarda 4 jogadores). Para a regra de pontuação compartilhada, bastaria somar os pontos das duas mãos ao final.

3 - Segurança e Chat (Pontos extras):

Para implementar o chat, você criaria uma nova thread no cliente só para input de texto e enviaria um pacote {'tipo': 'MSG', 'texto': '...'}. O servidor retransmitiria para todos.

A criptografia (ponta a ponta) exigiria bibliotecas como cryptography para gerar chaves assimétricas, o que pode fugir do escopo de um código simples com pickle, mas vale mencionar na documentação do trabalho.
